ARCHITECTURE — MEDALLION BRONZE / SILVER / GOLD
================================================================
Sources CSV (data/raw/)
    ↓ BronzeLoader (Python POO)
bronze.employees | bronze.absences | bronze.payroll | bronze.evaluations
    ↓ SilverTransformer (nettoyage, typage, validation)
silver.employees | silver.absences | silver.payroll | silver.evaluations
    ↓ GoldLoader (surrogate keys, jointures, SCD Type 2)
gold.Dim_Temps | gold.Dim_Departement | gold.Dim_Poste | gold.Dim_Ville
gold.Dim_Employe (SCD Type 2) | gold.Fact_Absences | gold.Fact_Paie
gold.Fact_Evaluations
    ↓
SSIS (orchestration) → SSAS Tabular (cube) → Power BI (dashboards)
    ↓
MLflow (scoring risque départ) → FastAPI → RAG LangChain/ChromaDB

DÉCISIONS TECHNIQUES IMPORTANTES (à ne pas remettre en question)
================================================================
1. SCD Type 2 sur Dim_Employe uniquement (autres dims = Type 1)
   Raison : Dim_Employe change (mutations, promotions) — historique crucial

2. DELETE FROM au lieu de TRUNCATE pour les dimensions Gold
   Raison : FK constraints empêchent TRUNCATE sur tables référencées

3. Ordre de suppression Gold (obligatoire) :
   Fact_Evaluations → Fact_Paie → Fact_Absences →
   Dim_Employe → Dim_Departement → Dim_Poste → Dim_Ville → Dim_Temps

4. Ordre d'insertion Gold (obligatoire) :
   Dims simples → Dim_Employe → Facts

5. Architecture SSIS : Execute Process Task appelle pipeline.py
   (pas de Foreach Loop — structures CSV trop différentes)

6. Power BI connecté via SSAS Tabular (pas directement SQL Server)
